| Раздел                 | Обоснование                                                                                                                                                                                                                                                                                                                                                                                                                                                                               | Какие проблемы закроет                                                                                                                                                                                                                             |
|------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Управление доступом    | В описании системы указано, что есть несколько точек регистрации клиентов, нет координации между ними, а пользователи иногда видят чужие ФИО в виде ошибки при вводе пароля. Имеются явные признаки проблем с идентификацией, аутентификацией и управлением учётными записями. Отсутствует единый провайдер идентичности (IDP) — система, которая хранит учётные записи и подтверждает личность пользователя для всех сервисов сразу. Нет ограничения доступа к системам обработки данных | нарушение контроля данных (несколько точек регистрации); ошибки с отображением чужих ФИО; отсутствие единого реестра учётных записей; отсутствие принципа минимальных привилегий                                                                   |
| Безопасность данных    | Приложение работает с большим объёмом персональных данных (ФИО, телефоны, паспорта, адреса, данные умного дома). При этом нет политики классификации данных, нет контроля над потоками персональных данных к партнёрам (УК), отсутствует Data Governance — набор правил, ролей и процессов, которые определяют, кто владеет данными, кто может их менять и как они перемещаются между системами                                                                                           | контроль внутренних потоков данных; ограничение на доступ к данным со стороны систем обработки (хранилище); предотвращение свободной передачи категорий данных (персональные данные) партнёрам                                                     |
| Защита инфраструктуры  | У компании есть firewall и собственный ЦОД, но архитектура не предполагает сегментации сети, контроля трафика между сервисами, регулярного сканирования уязвимостей. Нет автоматизированных проверок безопасности облачной инфраструктуры (компания использует облачные ресурсы для тестирования)                                                                                                                                                                                         | отсутствие сегментации сети между группами сервисов (продажи, ЖКУ, финансы, дата); неизолированность критических сервисов (онлайн-сделка, умный дом); риск перемещения злоумышленника по всей сети после взлома одного сервиса                     |
| Управление инцидентами | Бизнес прямо указал потребность: «настроить алертинг и автоматическое реагирование на подозрительные действия по логам приложения». Также отсутствует план реагирования на инциденты, не ведутся учения, нет статистики инцидентов                                                                                                                                                                                                                                                        | отсутствие алертинга и автоматического реагирования; невозможность отследить подозрительные действия (например, подбор паролей); отсутствие возможности проанализировать статистику инцидентов и корректировать политики безопасности              |
| Специфические аспекты  | Компания активно развивает мобильное приложение для собственников (умный дом, оплата ЖКУ), использует облачные ресурсы для тестирования и временного расширения мощностей, имеет собственный ЦОД. Это требует оценки физической безопасности серверных помещений, управления мобильными устройствами сотрудников (MDM), защищённого удалённого доступа (VPN) и политик резервного копирования в облаке                                                                                    | отсутствие контроля над мобильными устройствами сотрудников и подрядчиков; незащищённость удалённого доступа к ЦОД; отсутствие политик автоматического резервного копирования в облаке; неизвестный уровень физической защиты серверных помещений  |
