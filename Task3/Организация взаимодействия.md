## Открытие домофона по звонку с мобильного приложения

1. Собственник нажимает кнопку «Открыть дверь» в мобильном приложении
2. Мобильное приложение отправляет запрос с JWT токеном
3. API Gateway: проверяет JWT; извлекает user_id; вызывает Smart Home Integration Service
4. Smart Home Integration Service: запрашивает из CRM собственников квартиры по user_id; сопоставляет с door_id; возвращает {allowed: true}
5. API Gateway формирует запрос к партнёрской платформе на открытие двери
6. Партнёрская платформа отправляет команду на физический домофон
7. Ответ возвращается по цепочке обратно
8. API Gateway логирует факт открытия (user_id, door_id, timestamp, result=success)


## Автоматическое открытие шлагбаума по камере

1. Камера на шлагбауме распознаёт номер автомобиля
2. Партнёрская платформа получает событие: car_plate_hash = "a1b2c3..."
3. Партнёр отправляет запрос в API Gateway PropDevelopment с car_plate_hash, barrier_id, timestamp
4. API Gateway проверяет API Key партнёра
5. Smart Home Integration Service проверяет по своей внутренней таблице привязку собственника к car_plate_hash и доступ к barrier_id
6. Smart Home Integration Service возвращает {allowed: true/false}
7. Партнёр открывает или не открывает шлагбаум

## Биометрическое открытие домофона

1. Камера на домофоне подъезда распознаёт лицо
2. Партнёрская платформа получает событие: face_id_hash = "a1b2c3..."
3. Партнёр отправляет запрос в API Gateway PropDevelopment с face_id_hash, entrance_id, house_id, timestamp
4. API Gateway проверяет API Key партнёра
5. Smart Home Integration Service проверяет по своей внутренней таблице привязку собственника к face_id_hash, house_id и доступ к entrance_id
6. Smart Home Integration Service возвращает {allowed: true/false}
7. Дверь домофона открывается в случае true/ не открывается в случае false (оповещает об открытии/закрытии голсом)
