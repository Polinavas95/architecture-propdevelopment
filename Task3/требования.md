| №  | Требование                                                                                                                                                            | Обоснование                                                                |
|----|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------|
| Б1 | Все запросы к партнёрской платформе проходят через API Gateway , а не напрямую из мобильного приложения                                                               | Предотвращает подмену клиентского приложения и прямой доступ к устройствам |
| Б2 | Партнёрская платформа не хранит ПДн собственников, а только идентификаторы: user_id (внутренний), car_plate_hash (хэш номера), face_embedding (биометрический вектор) | Минимизация ущерба при утечки информации у партнёра                        |
| Б3 | Биометрические данные (если передаются) хранятся у партнёра в виде эмбеддингов с солью, без возможности восстановления при перехвате                                  | Соответствие законодательству о биометрии                                  |
| Б4 | Все запросы на открытие/закрытие дверей и контроль освещения логируются в SIEM PropDevelopment с полем action (open_door, raise_barrier) и result (success/denied)    | Обеспечение алертинга (потребность бизнеса)                                |
| Б5 | Партнёрская платформа не может инициировать запросы к PropDevelopment                                                                                                 | Предотвращение обратных атак через партнёра                                |
| Б6 | При передаче номера автомобиля на шлагбаум передаётся только хэш номера, а не сам номер                                                                               | Защита персональных данных                                                 |
